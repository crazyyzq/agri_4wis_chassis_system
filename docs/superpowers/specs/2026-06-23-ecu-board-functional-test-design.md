# ECU 板级全功能测试固件设计

日期：2026-06-23  
状态：已完成分节评审，等待用户审阅书面规格

## 1. 目标

为 `ECU_Isolation` V1.0 开发一套可重复使用的板级测试固件，用同一固件完成多块样板的非破坏性功能验证，并形成可追溯测试记录。

测试必须满足：

- 上电安全，任何异常退出均恢复安全状态。
- 支持单项调试和整板顺序测试。
- 每个硬件模块可独立执行和定位故障。
- 测试结果同时适合人工阅读和 PC 程序解析。
- 使用 HPM SDK 1.11.0、SEGGER Embedded Studio 8.28 和 J-Link。

## 2. 范围

### 2.1 必测功能

- 12 V 标称供电及板上电源轨。
- 上电、外部复位、软件复位、看门狗复位和安全默认状态。
- RGB LED。
- 片上 RAM、32 MB SDRAM、8 MB QSPI Flash、AT24C02 EEPROM。
- 4 路板载模拟输入。
- 12 路隔离数字输入。
- 12 路隔离低边数字输出。
- 4 路 Classic CAN 和 4 路软件可控终端电阻。
- 3 路隔离 RS485。
- 4 路隔离 RS232。
- 1 路隔离 SBUS 输入。
- 隔离调试串口和 JTAG/J-Link 调试链路。
- 正常、失败、超时、中止和复位路径的安全清理。

### 2.2 明确不在本阶段范围内

- CAN-FD。
- Ethernet。当前无连接设备，记录为 `SKIP_NO_DEVICE`，不影响整板 PASS。
- 外置 ADC 模块的 Modbus 协议。
- 伺服驱动器 CANopen、BMS、DCDC、逆变器和遥控器的设备联调。
- 液压系统、执行机构和整车控制。
- 欠压、过压、短路、EMC、温升、振动、老化等边界或可靠性试验。

## 3. 设计依据

- `doc/ECU/SCH_ECU_Isolation_原理图.pdf`，V1.0，2026-03-06。
- `doc/ECU/HPMicro-MR6750-L管脚对应表.xlsx`。

- HPM SDK 1.11.0。

引脚表工作表名称为 `MR6450-L`，与文件名不一致；本设计以原理图网络名和已成功编译的板级管脚配置为准。

## 4. 总体架构

系统分为四层：

1. **PC 终端**：操作员通过隔离调试串口选择测试、输入板卡编号、确认接线并保存日志。
2. **Test Runner**：负责命令解析、测试注册、依赖检查、状态机、超时、中止和结果汇总。
3. **测试模块**：Memory、ADC、DI、DO、CAN、RS485、RS232、SBUS、RGB、Boot/Safety。
4. **外部台架**：电源、仪器、回环线束、假负载和 USB 总线适配器。

测试模块只能通过 BSP 和 `safety_manager` 访问硬件。危险资源不能由测试模块直接写 GPIO。

## 6. 固件组件

### 6.1 `app_main`

- 调用板级最小初始化。
- 立即调用安全状态初始化。
- 初始化调试串口和 Test Runner。
- 进入裸机事件循环。

### 6.2 `cli`

提供：

- `list`
- `run <id>`
- `run all`
- `abort`
- `status`
- `report`
- `board <serial>`
- `help`

启动信息包含固件版本、板级名称、SDK 版本、复位原因、主要时钟和安全状态。

### 6.3 `test_runner`

- 维护测试注册表、必测/可跳过属性和依赖。
- 支持单项模式和整板模式。
- 控制测试生命周期与超时。
- 保存当前会话中的全部尝试，不用重试结果覆盖原失败记录。
- 汇总整板结果。

### 6.4 `safety_manager`

- 是 DO、CAN 终端和 RS485 DE 的唯一控制入口。
- 任意时刻最多允许一路 DO 开启。
- 跟踪所有受控资源当前状态。
- 提供幂等的 `safe_all_off()`，可从任何状态重复调用。

### 6.5 `result_writer`

- 输出面向操作员的文本。
- 每次测试额外输出一行 JSONL 结构化结果。
- ECU 记录启动后毫秒数；绝对时间由 PC 串口日志提供。

### 6.6 `test_limits`

集中定义所有电压、误差、通信速率、循环次数和超时，测试模块不得散布魔数。

### 6.7 测试模块

每个模块实现统一接口：

```text
prepare(context)
execute(context)
cleanup(context)
```

`cleanup` 无论成功、失败、超时或中止都必须执行。

## 7. 状态机与结果模型

### 7.1 运行状态

`IDLE → PREPARE → RUNNING → CLEANUP → RESULT → IDLE`

- `IDLE`：全部危险输出关闭。
- `PREPARE`：检查依赖、参数和操作员确认。
- `RUNNING`：执行测试并检查超时、中止标志和看门狗。
- `CLEANUP`：无条件恢复安全状态。
- `RESULT`：记录本次结果并更新会话汇总。

调试 UART RX 中断识别中止请求。长时间测试必须周期性查询中止标志并喂看门狗。

### 7.2 单项结果

- `PASS`：已执行且所有指标满足。
- `FAIL`：已执行但至少一个指标不满足。
- `SKIP`：已批准的非必测项未执行。
- `BLOCKED`：必测项因依赖失败、工装缺失或配置错误而未执行。

### 7.3 整板结果

- `PASS`：全部必测项 PASS，只存在已批准的 SKIP。
- `FAIL`：任一必测项 FAIL。
- `INCOMPLETE`：无 FAIL，但存在必测项 BLOCKED 或未执行。
- `ABORTED`：操作员中止整板流程。

## 8. 安全设计

- 上电首先关闭 12 路 DO 和 4 路 CAN 终端，并将 RS485 切回接收状态。
- 所有外部动作测试必须由操作员显式启动。
- DO 逐路测试，ON 时间不超过 500 ms，禁止并发开启。
- 参数越界在接触硬件前拒绝执行。
- 正常完成、错误、超时、`abort` 和软件复位均调用 `safe_all_off()`。
- 看门狗复位后仍先执行安全初始化，再输出复位原因。
- 手工确认超时记为 BLOCKED，不能默认 PASS。

## 9. 测试执行顺序

1. 断电预检：外观、连接器、BOOT 拨码、12 V 输入阻抗和工装接线。
2. 安全启动：限流上电、调试串口、复位原因、时钟、RGB 和安全默认状态。
3. 内部资源：片上 RAM、QSPI Flash、SDRAM、EEPROM。
4. 低能量 I/O：DI、ADC、SBUS。
5. 通信接口：RS232、RS485、Classic CAN。
6. 受控输出：DO 逐路低电流带载测试。
7. 汇总与复位：生成报告、关闭资源、软件复位并复查安全启动。

供电或安全启动失败时停止全部后续测试。其他独立模块失败不阻止无依赖测试继续执行。

## 10. 台架与工装

- 12 V 可调电源限流供电。
- 万用表和示波器测量电源、输出和通信波形。
- DI/DO 回环：DI 正端接 12 V，DI 负端通过对应 DO 下拉。
- DO 使用约 220 Ω、2 W 假负载接 12 V。
- ADC 使用可调信号源。
- CAN、RS485、RS232 使用对应 USB 适配器逐口测试。
- SBUS 使用遥控接收机或专用帧发生器。
- J-Link 负责下载和调试；隔离调试串口负责 CLI 与日志。

DI/DO 回环只验证逻辑链路，不能替代 DO 假负载电压测量。

## 11. 详细测试与验收标准

### 11.1 PWR：供电

- 测试电源设为 12.0 V 并启用限流。
- MCU_5V、ISO_5V：4.75～5.25 V。
- MCU_3V3、MCU_3V3A、ISO_3V3：3.135～3.465 V。
- 不允许异常升温、掉电或周期性重启。
- 首批样板只记录输入电流和纹波。获得三块正常样板的数据后，以统计结果和器件设计余量建立硬阈值。

### 11.2 BOOT/SAFE：启动、复位与安全状态

- 上电 3 s 内出现完整调试日志。
- 外部复位、软件复位和看门狗复位均能重新启动并识别复位原因。
- 每次启动后 DO 关闭、CAN 终端关闭、RS485 为接收状态。
- `abort`、测试超时和驱动错误后复查相同安全状态。

### 11.3 LED：RGB

- 红、绿、蓝分别亮、灭，不允许串色或无法关闭。
- 由操作员在 CLI 中确认。

### 11.4 MEM：存储器

片上 RAM：

- 仅测试专用缓冲区，不覆盖栈、堆、向量表或运行数据。
- 使用地址模式、`0x00`、`0xFF`、`0xAA`、`0x55` 和 walking-bit。

SDRAM：

- 分块测试未被程序占用的 SDRAM 区域。
- 使用固定模式、地址模式和地址线混叠检查。
- 每个写入值必须完整回读一致。

QSPI Flash：

- 读取 JEDEC ID。
- 校验当前固件镜像 CRC。
- 固定保留 XPI0 最后 64 KiB，即 `0x807F0000～0x807FFFFF`，作为测试区域。
- 链接配置必须保证应用镜像不能进入该区域。
- 写擦操作从 RAM 执行；测试后区域保持擦除态。
- Flash 几何参数与 8 MB 配置不一致时返回 BLOCKED，不执行擦除。

AT24C02 EEPROM：

- 固定使用 `0xF0～0xFF` 作为测试区域。
- 断电测试前，把 `0xF0～0xFF` 原值和恢复标记写入已保留的 QSPI 测试区，再写入 EEPROM 测试模式并回读。
- 重新上电后，启动代码根据恢复标记验证 EEPROM 保持性，恢复并核对原值，最后擦除 QSPI 恢复标记。
- 恢复过程被再次断电时，下一次启动继续恢复，不能遗留被覆盖的 EEPROM 原值。

### 11.5 ADC：4 路模拟输入

通道映射：

- EX1 → ADC3 CH6 / PF09。
- EX2 → ADC3 CH4 / PF05。
- EX3 → ADC3 CH5 / PF07。
- EX4 → ADC3 CH7 / PF10。

模拟前端比例：

`Vadc = Vin × 100 / (56 + 100) = Vin × 0.6410256`

| 外部输入 | ADC 引脚理论电压 |
|---:|---:|
| 0.0 V | 0.000 V |
| 0.5 V | 0.321 V |
| 2.5 V | 1.603 V |
| 5.0 V | 3.205 V |

- 每点多次采样并记录平均值、最小值、最大值和峰峰抖动。
- 外部量程换算误差不超过 ±2%。
- 检查单调性和通道间明显串扰。
- 本阶段不做过压和输入阈值极限测试。

### 11.6 DI：12 路隔离输入

- 每路分别施加 0 V 和 12 V。
- 验证无效和有效状态，连续采样后状态应稳定。
- 只做标称电压验证，不扫描输入阈值。

### 11.7 DO：12 路隔离低边输出

- 使用约 220 Ω、2 W 假负载接 12 V，测试电流约 55 mA。
- 逐路执行 OFF→ON→OFF。
- ON 最长 500 ms；任意时刻只允许一路开启。
- ON 时输出端不高于 0.5 V；OFF 时输出端应恢复接近 12 V。
- DI/DO 回环另外验证对应逻辑映射。

### 11.8 CAN：4 路 Classic CAN

- 速率：500 kbit/s。
- 使用外部 USB-CAN 逐口测试，不能只用 ECU 内部控制器互发替代。
- 覆盖标准帧、扩展帧、DLC 0 和 DLC 8。
- 数据覆盖 `0x00`、`0xFF`、`0xAA/55` 和递增模式。
- 每路双向各 1000 帧，ID、长度和数据全部一致，错误计数不增长。
- 拔插总线后可重新初始化并恢复，不做 H/L 短路注入。

CAN 终端：

- 测量前断开外部总线和适配器终端。
- 关闭时应为高阻。
- 开启时 CANH-CANL 为 115～125 Ω。
- 测试完成后必须关闭。

### 11.9 RS485：3 路

- 115200 bit/s、8N1。
- 覆盖不同帧长和 `0x00`、`0xFF`、`0xAA/55`、递增数据。
- 每路双向各 1000 帧，帧 CRC 和数据全部正确。
- 每次发送结束后 DE 必须恢复接收状态。
- 本阶段使用原始测试协议，不实现外置 ADC Modbus。

### 11.10 RS232：4 路

- 115200 bit/s、8N1。
- 数据模式和循环次数与 RS485 相同。
- 每路双向各 1000 帧无错误。
- 示波器抽查每路 TX 的正负电平转换。

### 11.11 SBUS

- 100000 bit/s、8E2，按板上反相电路后的 MCU 逻辑解析。
- 连续接收至少 100 个合法帧。
- 验证通道值、lost-frame 标志、failsafe 标志和断流超时。

### 11.12 DEBUG/JTAG

- 调试串口 115200 bit/s、8N1，命令、日志和中止输入可靠。
- J-Link 能执行擦写、下载、复位、暂停、单步和寄存器读取。

### 11.13 ETH

- 返回 `SKIP_NO_DEVICE`。
- 不初始化 PHY，不参与整板 PASS 判定。

## 12. 日志格式

每项结果至少包含：

- 板卡编号。
- 固件版本和 SDK 版本。
- 测试编号与尝试序号。
- 开始/结束的 ECU uptime。
- PASS、FAIL、SKIP 或 BLOCKED。
- 实测值、单位、上下限。
- 错误码和备注。

示例结构：

```json
{"board":"ECU-001","test":"ADC.EX1.2V5","attempt":1,"status":"PASS","value":2.49,"unit":"V","low":2.45,"high":2.55,"elapsed_ms":210}
```

## 13. 错误码原则

错误码按模块分段：

- `0x01xx`：Runner/参数/依赖。
- `0x02xx`：Power/Boot/Safety。
- `0x03xx`：Memory。
- `0x04xx`：ADC。
- `0x05xx`：DI/DO。
- `0x06xx`：CAN。
- `0x07xx`：RS485/RS232。
- `0x08xx`：SBUS。
- `0x09xx`：Debug/JTAG。

驱动层状态寄存器、错误计数和发生阶段必须写入备注字段，不能只返回通用失败。

## 14. 固件自身验证

### 14.1 PC 单元测试

- CLI 命令解析。
- 状态机迁移。
- 依赖和整板结果判定。
- 超时与中止处理。

### 14.2 目标板自动测试

- GPIO 默认状态。
- 复位原因和看门狗路径。
- 驱动错误与清理路径。

### 14.3 台架 HIL

- 电源、ADC、DI/DO、CAN、RS485、RS232 和 SBUS。
- 每块样板执行完整流程并保存日志。

### 14.4 发布门槛

- 固件成功编译链接。
- PC 单元测试全部通过。
- 安全启动冒烟测试通过。
- 整板必测项全部 PASS；ETH 是唯一预先批准的 SKIP。

## 15. 构建与可重复性

- 提供 PowerShell 构建脚本，固定 SDK 路径、板路径、应用路径和 `flash_sdram_xip` 构建类型。
- 提供 SES 工程生成脚本，不依赖操作员在 GUI 中重复选择路径。
- 构建输出放入独立 `build/` 或 `tmp/`，不污染源码目录。
- 固件版本、构建类型和 Git 提交号在启动日志中输出；工作空间纳入 Git 后启用提交号字段，未纳入前输出 `unversioned`。

## 16. 完成条件

满足以下条件即认为本阶段完成：

1. 测试固件和固定构建入口可由 SES/J-Link 下载运行。
2. 单项与整板命令均可工作。
3. 所有必测模块实现并按本规格验收。
4. 所有退出路径恢复安全状态。
5. 每块样板生成可解析测试记录。
6. PC 单元测试、目标板测试和台架整板测试通过。
7. Ethernet 保留为唯一批准的 SKIP；CAN-FD 不属于测试范围。
